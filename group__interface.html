int cc_read_all ( struct cc *  c,
struct cc_data *  data 
)

Read multiple types of data simultanously.

Use this function if you need to read multiple types of data (e.g. odometry and rplidar).

If you care only about single data type use one of convinience functions instead.

Function will block waiting for data unless last call returned CC_DATA_PENDING. Timeout with return value CC_ERROR and errno EAGAIN indicates device is not sending data types for some reason.

Data types with 0 size in data parameter are discarded silently without parsing.

Parameters
cpointer to internal library data
datauser supplied arrays with sizes
Returns
  • CC_OK indicates user arrays in data parameter were filled
  • CC_DATA_PENDING indicates at least one array in data parameter was filled completely and more data is pending (without blocking)
  • CC_ERROR indicates error, query errno for the details
